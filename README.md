# API Security Risk Analysis

## Internship
Future Interns Cyber Security Internship (2026)

## Task
Task 3 - API Security Risk Analysis

## Objective
To analyze a public API and identify security risks related to authentication, authorization, and data exposure.

## API Tested
https://jsonplaceholder.typicode.com/users

## Tools Used
- Browser
- Postman (optional)
- Developer Tools

## Observations
- API accessible without authentication
- User data exposed publicly
- No rate limiting detected
- No authorization controls in place

## Security Risks Identified
- No Authentication (Medium Risk)
- Data Exposure (Medium Risk)
- No Rate Limiting (Low Risk)

## Recommendations
- Implement authentication (API keys or JWT)
- Restrict sensitive data exposure
- Add rate limiting mechanisms
- Enforce proper authorization controls

## Outcome
Understood basic API security risks and mitigation strategies.
